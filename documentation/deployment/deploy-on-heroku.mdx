# Deploy to Heroku

![Heroku Dialog](/img/docs/deployment-heroku.png)

[Heroku](https://www.heroku.com/) is the easiest way to get started with **Jitsu**. If you don't have an active Heroku account you can [signup here](https://signup.heroku.com/)!

### Getting Running on Heroku

* Pick a destination for your event data, we recommend starting with [Postgres](https://devcenter.heroku.com/articles/heroku-postgresql).
* Click our [Deploy](https://heroku.com/deploy?template=https://github.com/jitsucom/jitsu) to Heroku link.
* Fill `GEO_MAXMIND_PATH` (optional) - if you are using maxmind for [geo data resolution](/docs/other-features/geo-data-resolution).

![Configuration Parameters](/img/docs/heroku-jitsu-configuration.png)

* Click **Deploy App** and you're alive!
* After your app is deployed, you can set up your API keys, Destinations and Source connectors on Jitsu Configurator UI [https://your_app.herokuapp.com/configurator](https://your-app.herokuapp.com/)

Once Jitsu is set up, place the following code on your web-app. (See our [JavaScript SDK 2.0 guide](/docs/sending-data/js-sdk) for fine-tuning):

```html
<script src="https://[your_app].herokuapp.com/s/lib.js" data-key="<YOUR_API_KEY_FROM_CONFIGURATOR_UI>" defer>
</script>
```
